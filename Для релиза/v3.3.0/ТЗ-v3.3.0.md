# Техническое Задание — CRM v3.3.0

## Цель релиза
Устранение критических архитектурных ошибок, найденных в ходе аудита: невалидные графы в БД,
частичная запись без транзакции, ненадёжная оркестрация LLM, утечки состояния UI.

## Тестирование

### Критерии приёмки
1. `npx tsc --noEmit` — 0 ошибок
2. Все fallback-графы проходят validateReactorGraph без port_kind_mismatch
3. При невалидном ответе LLM — retry выполнен, пользователь видит уведомление
4. Запись трёх графов — атомарная (при ошибке на 2-м ни один не записан)
5. История стилей сбрасывается при переходе между продуктами

### Тест-сценарии
1. Создать продукт через AI-compose → проверить граф в БД через DevTools
2. Открыть маску продукта A → изменить стили → открыть маску продукта B → кнопка Undo должна быть неактивна
3. Обновить ноду mask-styles-root через AI-чат → стили должны применяться в UI немедленно
4. Запустить Reactor с невалидным AI-провайдером → увидеть "AI вернул невалидный ответ"
