# Участие в разработке

Спасибо за интерес к JBrealty CRM. Ниже — краткие правила для изменений в репозитории.

## Как предложить изменение

1. Создайте issue с описанием задачи или бага (если изменение крупное).
2. Сделайте fork или ветку от `main`.
3. Внесите изменения, проверьте локально:
   ```bash
   npm install
   npm run build
   npm run dev
   ```
4. Оформите pull request с понятным заголовком и описанием «что» и «зачем».

## Стиль кода

- TypeScript/React в `src/`, без лишних абстракций.
- Следуйте существующему стилю файлов (отступы, именование, Tailwind-классы).
- Не коммитьте `node_modules`, `dist`, `.env` и секреты.

## Сообщения коммитов

Используйте понятные сообщения на русском или английском, например:

- `fix: исправлен переход лида между этапами`
- `feat: добавлен экспорт лидов в CSV`

## Вопросы

Обсуждение — через [Issues](https://gitverse.ru/Corpuscul/JBrealty-CRM/issues) репозитория.
