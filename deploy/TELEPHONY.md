# Телефония Билайн и 152-ФЗ

## Билайн бизнес (облачная АТС)

**Настройки → Каналы → Телефония → Билайн бизнес**

### Шаги в cloudpbx.beeline.ru

1. **Услуги → Интеграции → Интеграция по API** (платная опция) — включить
2. **Настройки → API** — создать токен
3. Скопировать **внутренний номер** сотрудника (или userId абонента)

### Шаги в CRM

1. Выбрать провайдер **Билайн бизнес**
2. Вставить **API-токен** и **внутренний номер**
3. Нажать **Сохранить**, затем **Подключить события Билайн**

CRM автоматически создаёт подписку XSI-Events. Билайн отправляет события на:

```
https://ваш-домен.ru/api/webhooks/telephony/beeline/null?secret=…
```

### Исходящие звонки

Из карточки сделки кнопка «Позвонить» вызывает API Билайн (`POST /abonents/{id}/call`), если указаны токен и внутренний номер.

### Записи разговоров

Ссылки на записи подставляются автоматически. Заголовок для скачивания:

`X-MPBX-API-AUTH-TOKEN: ваш_токен`

(заполняется автоматически при сохранении настроек Билайн)

## Привязка звонков к карточкам

| Настройка | Поведение |
|-----------|-----------|
| **Цеплять к активной сделке** (по умолчанию) | Звонок попадает в открытую сделку с этим номером (не «Сделка» / «Отказ») |
| **Обратная настройка** | Всегда к последней карточке по номеру |
| **Создавать карточку** | Если номера нет — новая сделка «Звонок +7…» |

Дублей не создаётся: один клиентский номер → одна активная сделка.

## Другие провайдеры

Webhook URL:

```
https://ваш-домен.ru/api/webhooks/telephony/{provider}
```

Провайдеры: `beeline`, `generic`, `mango`, `zadarma`, `uis`, `asterisk`.

## 152-ФЗ

- Запись звонков — отдельное согласие в политике
- Договор поручения с оператором связи
- При обезличивании лида номера в журнале звонков очищаются
